### L2 Proxy — MITM снифер для Lineage 2 C1 (Protocol 419)

Прокси-перехватчик трафика между клиентом и сервером Lineage 2.
Расшифровывает Blowfish (логин-сервер) и XOR (гейм-сервер) пакеты в реальном времени.

> Прокси не модифицирует трафик — только пассивно читает.

------

#### Использование

```bash
nano config.yaml

go run .
```

<br>

#### Конфиг (`config.yaml`)

```yaml
login:
  local: "127.0.0.1:2106"
  remote: "192.168.1.100:2106"

game:
  local: "127.0.0.1:7777"
  remote: "192.168.1.100:7777"

logging:
  enabled: true
  filename: "packets.log"
```

<br>

#### Флаги (переопределяют конфиг)

--config <path>        | путь к конфигу (по умолч. config.yaml)
--login-local <addr>   | адрес логин-прокси
--login-remote <addr>  | куда форвардить логин
--game-local <addr>    | адрес гейм-прокси
--game-remote <addr>   | куда форвардить гейм
 --out <file>          | файл для лога пакетов

<br>

#### Как это работает

1. Клиент подключается к `127.0.0.1:2106` / `127.0.0.1:7777`
2. Прокси форвардит все байты на реальный сервер **без изменений**
3. Копия каждого пакета расшифровывается для отображения в логе
4. **Логин (2106)**: Blowfish со статическим ключом
5. **Гейм (7777)**: XOR с динамическим ключом (захватывается из handshake)
